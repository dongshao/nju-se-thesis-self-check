# 学术型论文自检 50 条

适用对象：算法/方法/实证研究类论文（提出方法 + 做实验/实证 + 给结论）。

格式说明同 `engineering-50.md`。

---

## 🔴 红线（5 条）

### ACA-RED-1 — 学术规范：研究行为
- **问题**：是否存在抄袭、剽窃、伪造数据/图表/结论
- **检查方法**：
  1. 抽查相关工作章节 3 段长描述，搜索原文比对
  2. 检查实验数据是否合理（精度 100%、改进幅度极大、与基线差异不可信地大）
  3. 检查表格数据是否前后一致（同一指标在不同表中数值不同）
- **失败模板**：在 P.{页码} 发现 {可疑情形：与公开资料雷同 / 实验数据可疑 / 数据前后矛盾}。江苏省评议细则把"抄袭、伪造数据"列为单项不合格，整篇论文一票否决。
- **修改建议**：彻底重写涉嫌段落；重新整理实验数据并附原始日志/代码仓库链接；存疑数字标注计算方式或来源。

### ACA-RED-2 — 文不对题
- **问题**：标题中的核心方法/问题/数据范围，是否能在正文找到对应工作
- **检查方法**：
  1. 标题切名词短语 → 排除"研究 / 方法 / 实证"等通用词
  2. 每个关键词 grep 全文命中数（特别是方法章和实验章）
  3. 命中 0 → 标题党
- **失败模板**：标题中的"{关键词}"在方法章和实验章命中 {n} 次，未见对应工作。
- **修改建议**：(1) 在方法章/实验章补充与该关键词对应的实质工作；或 (2) 修改标题，使其与正文实际工作严格匹配。

### ACA-RED-3 — 正文篇幅
- **问题**：第一章至末尾章总页数 ≥ 50 页
- **检查方法**：从骨架元信息计算
- **失败模板**：正文实际篇幅约 {X} 页，未达 NJU 软院 50 页底线。
- **修改建议**：补足方法、实验、案例研究的细节；不要靠堆砌公式或扩大图表凑页数。

### ACA-RED-4 — 方法章与实验章工作量饱满度
- **问题**：方法 + 实验相关章节合计 ≥ 35 页（对应工程型的"研发工作量"）
- **检查方法**：
  1. 识别"方法/模型设计 / 实验 / 评估 / case study / 实证分析"等章节
  2. 把这些章总页数加起来
  3. 合计 ≥ 35 页 → ✅；合计 < 35 页 → 红线
- **失败模板**：方法/实验相关章节（{识别到的章节}）合计 {X} 页，少于 35 页。这通常被评审视为"研究工作量不饱满"。
- **修改建议**：补充：(1) 方法的更详细推导/伪代码；(2) 更多对照实验；(3) 消融实验；(4) 定性 case study。

### ACA-RED-5 — 模型/算法描述完整性
- **问题**：是否有清晰的"问题形式化 + 模型/算法设计 + 实验设计"三段
- **检查方法**：
  1. grep 第 2-4 章是否含"问题描述""问题形式化""定义""算法 X""伪代码""实验设置"
  2. 任一缺失 → 红线
- **失败模板**：论文缺少 {问题形式化 / 算法详细设计 / 实验设置} 章节，方法学结构不完整。
- **修改建议**：必备三段：(1) 问题描述（含输入、输出、目标函数/优化目标、约束）；(2) 方法详细设计（含图示、伪代码、复杂度分析）；(3) 实验设置（数据集、基线、指标、超参数、硬件环境）。

## 学术质量（30 条）

### ACA-A-01 — 摘要四要素完整性
- **问题**：摘要是否包含"问题 + 方法 + 关键实验 + 主要结果"
- **检查方法**：读摘要全文，逐项核对 4 要素
- **失败模板**：摘要主要论述"{X}"，缺少 {缺失的要素}。
- **修改建议**：摘要 600-900 字。建议结构：(1) 1-2 句问题与现有方法不足；(2) 1-2 句本文方法核心思想；(3) 2-3 句关键实验设置（数据集、基线）；(4) 2-3 句主要结果（具体指标提升数字）。

### ACA-A-02 — 第一章问题描述清晰度
- **问题**：第一章是否给出**清晰可验证**的研究问题，而非"随着……发展"开头
- **检查方法**：读第一章前 3 页，检查是否含可证伪的研究问题（RQ）
- **失败模板**：第一章问题描述以行业泛述为主（P.{页码}），缺少明确的研究问题陈述。
- **修改建议**：用 1-3 个 RQ（Research Question）形式提出本文要解决的问题，每个 RQ 都应在实验章节有对应回答。

### ACA-A-03 — 创新点声明明确
- **问题**：是否在绪论或末尾章列点声明 1-3 条独立贡献，每条可被验证
- **检查方法**：grep "贡献""contributions""创新点"在第一章和末尾章
- **失败模板**：未见明确列点的贡献声明；或贡献描述空泛（如"提出了一种新方法"）。
- **修改建议**：每条贡献按"我们提出了 X，相比 Y 在 Z 方面有改进，通过 W 实验验证"格式。

### ACA-A-04 — 创新点的边界与独立性
- **问题**：若有多个组件创新点，每个组件相对已有方法的差异是否独立可验证
- **检查方法**：对照创新点声明 → 检查方法章是否有对应小节 → 检查实验章是否有对应消融实验
- **失败模板**：声明了 {n} 条创新点（如"双向语义对齐 / 动静特征分离 / 两阶段检索"），但消融实验未分别验证各点的独立贡献。
- **修改建议**：补充消融实验：每次去掉一个组件，对比指标变化，证明每个创新点的独立价值。

### ACA-A-05 — 第二章相关工作组织（按主题/路线）
- **问题**：相关工作是否按主题/技术路线分类组织，而非按时间或简单堆砌
- **检查方法**：抽查第二章二级标题，看是否按方法路线分类
- **失败模板**：第二章为时间序的文献堆砌（P.{页码}），缺少按主题/路线的分类组织。
- **修改建议**：第二章重组为 2-4 个主题小节，每节内按相似方法分组讨论，节末总结该路线的局限引出本文工作。

### ACA-A-06 — 与最新 SOTA 对比覆盖
- **问题**：是否引用并对比了近 1-2 年代表性工作（顶会、顶刊、知名预印本）
- **检查方法**：抽样最近的 5-8 篇文献年份，统计 2024-2025 年文献占比
- **失败模板**：参考文献近 1-2 年（2024-2025）的工作仅 {n} 篇，对最新 SOTA 覆盖不足。
- **修改建议**：补充近期顶会工作（如 NeurIPS / ICML / CVPR / ACL / ICSE / FSE / ASE 等近一年）作为基线或对照。

### ACA-A-07 — 问题形式化
- **问题**：算法/模型类论文是否有明确的输入、输出、目标函数/优化目标、约束的形式化定义
- **检查方法**：grep 方法章"定义""问题形式化""输入""输出""目标函数""约束"
- **失败模板**：方法章缺少形式化的问题定义（无明确输入输出表示、无目标函数/约束）。
- **修改建议**：在方法章开头加 1 节"问题定义"：用数学符号定义输入空间、输出空间、目标函数、约束条件。

### ACA-A-08 — 方法描述详实度
- **问题**：方法描述是否包含模型架构图 + 关键组件详述 + 关键算子推导
- **检查方法**：抽查方法章是否有架构图、是否每个组件都有详述、是否有数学推导
- **失败模板**：方法章对 {X} 组件描述过于简略（仅 1-2 段，无图无公式）。
- **修改建议**：每个核心组件至少包括：(1) 设计动机；(2) 架构图或数据流图；(3) 关键算子的数学定义；(4) 与已有方法的差异点说明。

### ACA-A-09 — 算法伪代码规范
- **问题**：方法章是否有清晰的伪代码（含输入、输出、变量定义、注释、复杂度分析）
- **检查方法**：grep "Algorithm""算法"在方法章；抽查是否有伪代码块
- **失败模板**：方法章缺少伪代码；或伪代码缩进/变量定义混乱（P.{页码}）。
- **修改建议**：用标准伪代码格式（如 algorithmicx LaTeX 包或 Word 算法环境）写至少 1 个核心算法的伪代码，含输入、输出、循环、条件、关键操作注释，末尾给时间/空间复杂度。

### ACA-A-10 — 数据集说明完整性
- **问题**：是否说明数据集来源、规模、划分、预处理、版权/伦理
- **检查方法**：grep "数据集""dataset"在实验章；检查是否含来源、规模、划分比例
- **失败模板**：数据集 "{X}" 描述缺少 {规模 / 划分 / 预处理 / 来源}。
- **修改建议**：每个使用的数据集补完整说明：来源（论文/链接）、规模（样本数、类别数）、划分（训练/验证/测试比例）、预处理步骤、是否平衡、版权/伦理是否合规。

### ACA-A-11 — 数据集偏差讨论
- **问题**：是否讨论了数据集的潜在偏差对结论的影响
- **检查方法**：grep "偏差""bias""局限"在实验或讨论章
- **失败模板**：缺少数据集偏差对结论有效性影响的讨论。在实证研究类论文中，这是评审高频追问点。
- **修改建议**：在实验章末尾或专门"威胁有效性"小节，至少讨论：(1) 数据筛选标准的潜在偏差；(2) 样本量是否充足；(3) 是否能推广到其他场景/数据集。

### ACA-A-12 — 评价指标完备性
- **问题**：评价指标是否充分覆盖任务的不同维度（不只 Top-K 或 Accuracy）
- **检查方法**：抽查实验章使用的指标列表
- **失败模板**：实验主要使用 {单一指标}，缺少 {该任务的其他重要指标}。
- **修改建议**：根据任务类型补充：(1) 分类任务：Precision / Recall / F1 / AUC；(2) 检索/排序：MAP / MRR / nDCG / Top-K；(3) 缺陷定位：Effort@K；(4) 代码生成：CodeBLEU / Pass@K；(5) 安全/隐私：False Positive Rate。

### ACA-A-13 — 基线选择合理性
- **问题**：基线是否包含同类强基线（SOTA），而非只比弱基线/旧方法
- **检查方法**：检查基线列表是否含近 1-2 年的代表性方法
- **失败模板**：基线主要为 {老方法 / 弱基线}，缺少近期同类 SOTA 作为对照。
- **修改建议**：补充至少 1-2 个近期 SOTA 作为基线；若 SOTA 因代码不开源等原因无法直接对比，至少在文中讨论性能差异并给出引用对照。

### ACA-A-14 — 实验设置可复现性
- **问题**：超参数、随机种子、硬件环境、训练时长是否完整给出
- **检查方法**：抽查实验章设置部分
- **失败模板**：实验设置缺少 {超参表 / 随机种子 / 硬件环境 / 软件版本}。
- **修改建议**：补一张超参数表（学习率、batch size、优化器、epoch 数等）；声明随机种子；列出 GPU/CPU 型号、CUDA 版本、关键依赖库版本。代码若开源，给链接。

### ACA-A-15 — 消融实验
- **问题**：是否有消融实验验证每个核心组件的独立贡献
- **检查方法**：grep "消融""ablation"在实验章
- **失败模板**：缺少消融实验，无法证明各创新点的独立贡献。
- **修改建议**：对每个创新组件做"去掉该组件 vs 保留"的对比；至少 3-5 个消融配置。

### ACA-A-16 — 统计显著性
- **问题**：关键实验结果是否多次运行报告均值/方差，并做显著性检验
- **检查方法**：grep "均值""方差""标准差""std""p-value""t-test""显著性"
- **失败模板**：实验结果未报告多次运行的均值/方差；或未做显著性检验。
- **修改建议**：核心对比实验运行 ≥ 3 次取均值±方差；与基线比较时做 t-test 或 Wilcoxon 检验，给出 p-value。

### ACA-A-17 — 横向对比表
- **问题**：是否有完整的横向对比表（同设置下各方法的全部指标）
- **检查方法**：抽查实验章是否有总对比表
- **失败模板**：缺少完整的方法横向对比表，难以一眼看出本文方法相对各基线的优势。
- **修改建议**：给一张总对比表，行是方法（含本文方法），列是各指标，最优值加粗，子最优加下划线。

### ACA-A-18 — 定性 case study
- **问题**：是否选 1-2 个典型案例做深度剖析（成功与失败各一个更佳）
- **检查方法**：grep "案例分析""case study""典型示例"
- **失败模板**：实验章侧重定量分析，缺少定性 case study。
- **修改建议**：选 1 个本方法成功而基线失败的案例，分析为什么本方法奏效；选 1 个本方法仍然失败的案例，分析失败原因。

### ACA-A-19 — 失败原因分析
- **问题**：失败/边界情况是否有原因分析（不只是报告失败比例）
- **检查方法**：grep "失败""bad case""未能"在实验章
- **失败模板**：报告了 {n}% 的失败率，但未对失败案例做归因分析。
- **修改建议**：将失败案例分类（如"数据问题 / 算法局限 / 工程缺陷"），每类选 1 个代表深入分析。

### ACA-A-20 — 泛化能力评估
- **问题**：是否在样本外/跨领域/跨数据集验证泛化能力
- **检查方法**：grep "泛化""generalization""跨数据集""out-of-distribution""零样本"
- **失败模板**：缺少跨数据集或样本外的泛化能力评估。
- **修改建议**：用至少 1 个本文未训练过的数据集测试方法的迁移性能，或做留一交叉验证。

### ACA-A-21 — 计算开销分析
- **问题**：是否报告训练时长、推理延迟、内存占用等开销
- **检查方法**：grep "训练时间""推理时间""GPU""显存""复杂度"
- **失败模板**：缺少训练/推理开销分析。
- **修改建议**：补充 (1) 训练总时长；(2) 单样本推理延迟；(3) 模型参数量与显存占用；(4) 与基线的复杂度对比。

### ACA-A-22 — 实证研究的内/外部效度
- **问题**：实证研究类论文是否专门讨论 internal validity / external validity / construct validity
- **检查方法**：判断是否实证研究（grep "实证""empirical""实证研究"）；若是 → grep "效度""validity""威胁"
- **失败模板**：本论文为实证研究，但缺少有效性威胁讨论（"Threats to Validity"）。
- **修改建议**：补一节 "Threats to Validity"，分内部/外部/构念三类讨论。

### ACA-A-23 — LLM 类研究的特殊维度（仅 LLM 类）
- **问题**：若涉及 LLM，是否报告幻觉率、prompt 设计策略、自动化校验机制
- **检查方法**：判断是否 LLM 类后 → grep "幻觉""hallucination""prompt"
- **失败模板**：本论文使用 LLM，但缺少对幻觉率、prompt 策略、自动化校验的论述。
- **修改建议**：补充 (1) prompt 模板与设计原则；(2) 多次采样的稳定性测试；(3) 输出后置校验机制；(4) 幻觉率定量评估。

### ACA-A-24 — 模型局限性诚实陈述
- **问题**：是否诚实陈述模型/方法的局限（不要写"完美无缺"）
- **检查方法**：grep "局限""不足""limitation"在末尾章
- **失败模板**：末尾章缺少对方法局限性的诚实陈述（仅有"取得了较好结果"等）。
- **修改建议**：末尾章补一节"局限与展望"，明确指出 (1) 方法在何种场景下表现下降；(2) 计算开销/数据需求等约束；(3) 未解决的问题。

### ACA-A-25 — 第二章每条相关工作均与本文方法有对照
- **问题**：第二章讨论的每条相关工作，是否在本文方法或实验中有对应处理
- **检查方法**：抽查第二章 5 条相关工作 → 检查方法章/实验章是否引用并对照
- **失败模板**：第二章 §{节号} 讨论的 "{工作名}" 在本文方法或实验中未被引用对照。
- **修改建议**：二选一：(1) 在方法章或实验章补充与该工作的对照；(2) 从第二章删除该工作的讨论。

### ACA-A-26 — 工程实现作为方法验证而非主体
- **问题**：工程实现章节是否作为"方法验证的工具"而非主体（学术型论文不应被工程实现压过）
- **检查方法**：检查方法章 + 实验章页数 vs 工程实现章页数比例
- **失败模板**：工程实现章节 ({X} 页) 比方法章 + 实验章合计 ({Y} 页) 还多，论文重心偏向工程而非学术贡献。
- **修改建议**：压缩工程实现章节（仅保留与方法验证相关的细节）；扩充方法章和实验章。

### ACA-A-27 — 末尾章展望与本文局限对应
- **问题**：末尾章展望是否与本文方法的局限直接呼应
- **检查方法**：对照"局限性"小节与"展望"小节
- **失败模板**：展望与本文方法局限脱节，仅给出泛泛"未来可应用于更多领域"。
- **修改建议**：每条展望对应一个具体局限（"局限 X → 下一步可通过 Y 解决"）。

### ACA-A-28 — 公式符号体系前后一致
- **问题**：方法章定义的符号，是否在后续章节保持一致
- **检查方法**：抽查 5 个核心符号 → grep 在第 3-5 章的命中是否一致
- **失败模板**：符号 "{X}" 在 P.{页码 1} 定义为 "{含义 1}"，但 P.{页码 2} 改为 "{含义 2}"。
- **修改建议**：在方法章建立符号表（Notation table），并保证全文一致使用；不重用同一字母表示不同含义。

### ACA-A-29 — 引用近期 arXiv/顶会论文
- **问题**：是否引用了近 1-2 年 arXiv 预印本或顶会论文作为对照（针对发展快的方向如 LLM、AI4SE）
- **检查方法**：判断方向是否快速演进；若是 → 检查近 1-2 年文献占比
- **失败模板**：本论文方向 ({X}) 演进较快，但近 1-2 年的代表性工作引用不足。
- **修改建议**：补充近期 NeurIPS / ICML / ACL / ICSE / FSE / ASE / arXiv 代表性工作；arXiv 引用注明发布日期与版本（arXiv 总量上限见 ACA-F-02：≤ 5 篇）。

### ACA-A-30 — 第二章篇幅与重心
- **问题**：第二章相关工作是否占合理篇幅（一般 6-12 页），不至于过短或挤占方法实验
- **检查方法**：从骨架计算第二章页数
- **失败模板**：第二章 {X} 页，{过短无法体现综述深度 / 过长挤占方法实验}。
- **修改建议**：调整到 6-12 页；过短的补充更多分类讨论；过长的精简到与本文工作直接相关的方向。
